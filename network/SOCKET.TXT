# 소켓 
    - 소켓을 통해 통신의 내부처리 과정 이해없이도 
    네트워크 프로그램 개발 가능

> 데이터 타입
    - 소켓은 생성과 , 설정 과정이 끝나면 운영체제의 통신관련 정보를 참조해
    다양한 작업을 편리하게 할 수 있는 데이터 타입


> 통신 종단점
    - 소켓은 통신의 출발점과 도착점 
    

> 네트워크 프로그래밍 인터페이스
    - TCP/IP프로토콜의 관점 
    - 통신 양단이 모두 소켓을 사용할 필요는 없다. 

    ICMP 
    - INTERNET CONTROL MESSAGE PROTOCOL 
    - 서버와 게이트웨이 사이에서 메시지를 제어하고 알려주는 프로토콜


    IGMP 
    - INTERNET GROUP MESSAGE PROTOCOL 
    - PC가 멀티캐스트로 통신할수 있다는것을 라우터에 통지하는 프로토콜 

> 윈도우 소켓 (WINSOCK)
    - 윈도우 환경에서 사용할수 있게 만든 네트워크 프로그래밍 인터페이스 

    특징 
        - DLL ( DYNAMIC LINK LIBRARY ) 을 통해 대부분의 기능 제공 
        - GUI를 갖추고 메시지 구동방식으로 동작 
        - 운영체제 차원에서 멀티스레드를 지원한다.


네트워크 프로그램 

> 경계 구분 

송신자 
- 항상 고정 길이 데이터를 보낸다
- 경계구분을 위한 특별한표시 EOR 
- 보낼데이터 길이를 고저길이 + 가변길이 데이터를 보낸다.

수신자
- 항상 고정길이 데이터를 받음
- EOR 이 나올때까지 데이터를 읽은후 처리 
- 고정길이 데이터를 읽어 뒤의 데이터길이를 알아낸다.
