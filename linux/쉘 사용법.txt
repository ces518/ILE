쉘의 기능 

# 쉘이란 ? 
- 사용자 인터페이스를 제공하는 프로그램

# 쉘이 기능 

1. 명령어 해석기 기능
    - 사용자와 커널 사이에 명령해석 전달
    - 입력한 명령어나 파일을 해석하고 명령을 수행하는기능.

2. 프로그래밍 기능

 - 쉘은 자체 내에 프로그래밍 기능이있어 프로그램 작성가능.
 - 여러 명령어를 반복적으로 수행하는 작업을 하나의 프로그램으로 만듬
 - 쉘프로그램을 쉘스크립트라고한다.

 3. 사용자 환경설정
  - 초기화 파일기능.
  - 명령을 찾아오는 경로, 새로운 파일의 기본권한
  - 다양한 환경변수 설정등 
  - 로그인시 초기화파일이 실행되어 사용자초기설정이 설정됨



 # 쉘 동작순서

 1. 실행
 2. 프롬프트표시
 3. 내장명령인지 -> 명령실행
 4. 자식 프로세스 생성 
 5. 부모쉘대기 
 6. 실행파일인가 ? -> 커널이 실행파일을 자식프로세스에서 실행후 복귀 
 7. 스크립트 끝인경우 자식프로세스에서 실행후 복귀



 # 쉘 종류

 1. 본쉘 
 - 최초의 쉘
 - 스티브본이 개발
 - 시스템 관리작업을 수행하는 쉘스크립트들은 본쉘기반
 - 히스토리기능, 앨리어스,작업제어.



 2. 콘쉘
 - 1983년 AT&T 연구소의 데이비드콘 이 발표
 - 히스토리, 앨리어스 등 C쉘의 특징도 모두제공, 처리속도 빠름


 3. C쉘
 - 버클리에서 빌조이가 개발
 - C언어와 같아 C쉘이라 불림
 - 크기가 커지고 속도가 느리지만 편리한 기능.
 - 앨리어스나 히스토리제공 


 4. 배시쉘
 - 본쉘을 기능으로 브라잉너폭스가 개발
 - C쉘,콘쉘의기능도 포함 


 5. 기타 쉘
 - TC
 - 데스크탑 콘쉘
 - z쉘
